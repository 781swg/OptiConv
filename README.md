# 🖼️ ImgProc-C
### High-Performance Image Processing Library in C11

[![Language](https://img.shields.io/badge/language-C11-blue.svg)](https://en.cppreference.com/w/c/11)
[![Build](https://img.shields.io/badge/build-CMake-green.svg)](https://cmake.org/)
[![License](https://img.shields.io/badge/license-MIT-lightgrey.svg)](LICENSE)
[![Memory](https://img.shields.io/badge/memory-Valgrind--safe-brightgreen.svg)](https://valgrind.org/)

**ImgProc-C** — это мощная и модульная библиотека на языке Си для низкоуровневой обработки изображений. Она сочетает в себе математическую точность алгоритмов компьютерного зрения с высокой производительностью, характерной для системного программирования. Проект включает в себя как саму библиотеку (API), так и готовую консольную утилиту (CLI).

---

## 🌟 Ключевые особенности

*   **🧩 Модульный дизайн**: Четкое разделение ответственности. Ядро (матричные вычисления), наборы фильтров и подсистема ввода-вывода (I/O) независимы друг от друга.
*   **🚀 Высокая производительность**: Алгоритмы спроектированы с учетом особенностей кэш-памяти процессора (Row-major access), что минимизирует задержки при обработке тяжелых изображений.
*   **🛡️ 100% Memory Safe**: В проекте реализована строгая обработка кодов возврата (Error Codes) для каждой аллокации. Утечки памяти отсутствуют (проверено через Valgrind).
*   **📷 Поддержка форматов**: Чтение и запись PNG, JPG, BMP и TGA «из коробки» благодаря легковесной интеграции с `stb_image`.

---

## 🛠 Реализованные алгоритмы

| Алгоритм | Описание | Сложность | Применение |
| :--- | :--- | :--- | :--- |
| **Generic Convolution** | Универсальный движок свертки с поддержкой Edge Clamping. | $O(W \cdot H \cdot K^2)$ | База для любых матричных фильтров. |
| **Gaussian Blur** | Размытие по Гауссу (генерация нормализованного ядра). | $O(W \cdot H \cdot K^2)$ | Устранение высокочастотного шума. |
| **Median Filter** | Поиск медианы в скользящем окне (пузырьковая сортировка). | $O(W \cdot H \cdot K^2 \log K)$ | Удаление импульсного шума ("соль/перец"). |

*(Где W, H — размеры картинки, K — размер ядра/окна).*

## ⚙️ Инструкция по сборке

### 1. Установка зависимостей
Для сборки проекта требуются компилятор Си, CMake и утилита Make (или Ninja).

**Для Ubuntu/Debian:**
```bash
sudo apt update
sudo apt install build-essential cmake valgrind doxygen
```
**Для macOS (через Homebrew):**
```bash
brew install cmake valgrind doxygen
```

### 2. Компиляция проекта
Клонируйте репозиторий и выполните сборку (Out-of-source build):

```bash
git clone https://github.com/yourusername/ImgProc-C.git
cd ImgProc-C
mkdir build && cd build
cmake ..
make
```
После компиляции в папке `build` появится исполняемый файл `imgproc_cli` и бинарные файлы тестов.

## 🖥️ Руководство пользователя (CLI)

Консольная утилита поддерживает строгий формат аргументов:
```bash
./imgproc_cli <input_file> <filter_flag> <output_file> [parameters...]
```

### Доступные фильтры и примеры запуска:

**1. Размытие по Гауссу (`-gauss`)**  
Требует 2 параметра: размер ядра (нечетное число) и значение `sigma` (сила размытия).
```bash
# Размытие с ядром 5x5 и сигмой 1.5
./imgproc_cli original.png -gauss result_blur.png 5 1.5
```

**2. Медианный фильтр (`-median`)**  
Требует 1 параметр: размер окна (нечетное число). Идеально для удаления битых пикселей.
```bash
# Очистка от шума окном 3x3
./imgproc_cli noisy.jpg -median clean.png 3
```

## 🧪 Тестирование и Анализ памяти

Проект покрыт юнит-тестами, проверяющими математическую точность и устойчивость к невалидным данным (`NULL` указатели, пустые файлы и т.д.).

### Запуск Unit-тестов
Находясь в папке `build`, вы можете запустить все тесты разом через CTest:
```bash
ctest --output-on-failure
```
Или запустить модули по отдельности для детального вывода:
```bash
./test_core
./test_filters
./test_io
```

### Проверка на утечки памяти (Valgrind)
Мы гарантируем корректную работу с динамической памятью. Чтобы убедиться в отсутствии утечек, запустите утилиту через Valgrind:
```bash
valgrind --leak-check=full --show-leak-kinds=all --track-origins=yes ./imgproc_cli ../examples/input.png -gauss out.png 3 1.0
```
**Ожидаемый результат в конце лога:**
`All heap blocks were freed -- no leaks are possible`

## 🤝 Код-стайл и стандарты
*   **Стандарт языка:** С11.
*   **Инкапсуляция:** Внутренние функции скрыты с помощью ключевого слова `static`.
